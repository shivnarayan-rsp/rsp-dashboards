First-Pass Framing — Read Before Using This dashboard represents a first structured view of MPS profitability at Ricoh South Pacific. Revenue figures are derived from BMS contract rates (service component only). Cost data — parts and consumables — is partial in this release: labour is estimated at NZD 49/hr × visit hours; consumable costs use the Toner Delivery → Order path where available. GM% figures are indicative, not audited. Use them to identify machines and territories that need investigation, not as a final financial record. Refinements will be made as the data model matures. If a number looks wrong, log it as an action and flag it for Vijendra.

⊕ GM Calculation

Gross Margin (GM%) is the primary profitability measure for each MPS machine and the portfolio. It is calculated as:

GM% = ( Revenue − Total Cost ) ÷ Revenue × 100 Where: Total Cost = Parts Cost + Labour Cost + Consumable Cost Revenue = CPC Revenue + Fixed Base Revenue (service component only)

All values are converted to NZD using the locked FX rates before GM% is computed. GM% is a blended rate — it does not separate finance margin from service margin.

$ Revenue Attribution

Single-machine contracts

Revenue is taken directly from the BMS contract rates for that machine's contract. The service component (CPC charges + fixed base) is used.

Multi-machine contracts

Where one contract covers multiple machines (e.g. 5 machines under one agreement), revenue is attributed using equal split in this release:

Revenue per machine = Total Contract Service Revenue ÷ Machine Count

A volume-weighted split (CPC revenue weighted by per-machine meter reads) is planned for a future release, once per-machine meter data is reliably joined. The equal split understates revenue for high-volume machines and overstates it for low-volume machines.

What is excluded from revenue

  • Finance/rental component of RP contracts (shown separately in BMS as a finance charge)
  • GST, VAT and equivalent local taxes
  • Ad-hoc call charges outside the contract
  • Hardware sale revenue (one-off, not recurring MPS)

↓ Cost Components

ComponentSourceMethodStatus
Parts Cost BMS — service_jobs → service_job_activities → service_job_lines Sum of buy_price_cents × used_quantity for service job lines linked to the asset's serial number, last 90 days Phase 1 — partial (join via activity_id chain)

Costs are averaged to a monthly figure by dividing 90-day totals by 3. This smooths seasonal variation but may not reflect the current month accurately.

◉ Consumable Cost Methodology

Phase 1 approach (current)

Consumable cost is sourced from Salesforce via the Toner Delivery case path:

Consumable Cost = SELECT SUM(OrderItem.TotalPrice) FROM Case WHERE Work_Type__c = 'Toner Delivery' AND Asset.SerialNumber = [machine serial] AND CreatedDate = LAST_N_DAYS:90 -- via Case.Order__c → Order → OrderItem
  • Only Toner Delivery cases are included — other consumable types (e.g. drums, waste toner) are excluded in Phase 1
  • The Work_Type__c field on the Case must be set to 'Toner Delivery' for the delivery to be captured
  • If a Toner Delivery case has no linked Order, it is not counted (data quality issue — log an action)
  • OrderItem.TotalPrice is in local currency; converted to NZD using the locked FX rates

Future enhancements

Phase 2 will include drums, waste toner bottles and other consumables tracked via separate Work_Type values or product codes on OrderItem.

⏱ Labour Rate Assumptions

A flat labour rate of NZD 49/hr is applied across all territories. This is derived from:

USD 35,000/yr ÷ 1,200 billable hrs/yr = USD 29.17/hr ÷ 0.595 (USD/NZD) = NZD 49/hr (rounded)

This rate is a proxy for the average Pacific field technician cost fully loaded (salary + oncosts). It does not vary by territory in Phase 1 — future releases may apply territory-specific rates.

How to update the labour rate

  1. Open the relevant .html file in a text editor
  2. Find labourRateNZD: 49 in the DATA object near the top of the <script> block
  3. Update the value (e.g. 52) and save
  4. All GM calculations on that page will recalculate automatically on next browser load
  5. If the dashboard is rebuilt from MCP at a later date, update the constant in the build instructions so it persists

≡ Contract Type Definitions

TypeFull NameDescriptionRevenue treatment
MP Managed Print Customer pays a per-click (CPC) rate for pages printed, with an optional minimum volume commitment. Ricoh provides the machine, service, and consumables. CPC revenue = volume × rate. Fixed base if applicable. Finance component excluded.
RP Rental Print Customer pays a monthly rental fee for the equipment plus a service/click component. The machine remains Ricoh's asset. Service CPC + fixed service base only. Rental (finance) component excluded from MPS GM calculation.

Both contract types are included in the MPS portfolio. The finance/rental component is not included in the GM calculation as it flows to a separate P&L line.

↔ FX Conversion

All monetary values are converted to NZD for portfolio comparison using locked static rates. These rates are reviewed monthly by Minoo and updated in the DATA.fxRates object in each file.

TerritoryCurrencyRate to NZDExample
RSP (NZ)NZD1.000NZD 100 = NZD 100

SamoaWST0.540WST 1,000 = NZD 540
VanuatuVUV0.012VUV 100,000 = NZD 1,200
TongaTOP0.640TOP 1,000 = NZD 640

VUV note: The Vanuatu Vatu has no decimal sub-unit. BMS stores VUV amounts as whole integer values. Unlike other currencies where BMS stores values in cents (÷100 to get the amount), VUV values are already in full Vatu. The dashboard applies × FX.VUV directly without dividing by 100.

How to update FX rates

Find fxRates: { NZD:1.000, SBD:0.180, ... } in the DATA object of each HTML file and update the relevant currency value. Refresh the browser to see recalculated NZD figures.

★ Major Account Threshold

An account is classified as a major account if it meets any one of these criteria:

  • Fleet size: 3 or more machines on active MPS contracts
  • Revenue: NZD 500 or more per month across all MPS contracts
  • Multi-territory: Active machines in 2 or more RSP territories

Major accounts receive quarterly (or biannual) QBR review tracked in mps_accounts in Supabase. The QBR tracker on the Contracts page flags accounts that are overdue for review.

⚠ Known Limitations (Phase 1)

Revenue attribution is equal-split for multi-machine contracts. A 5-machine contract assigns 1/5 of revenue to each machine. Machines with high volume will look more profitable than they are; low-volume machines will look less profitable.
Parts cost join chain is complex. The BMS path is service_jobs → service_job_activities → service_job_assets (via activity_id) → service_job_lines. If any link in this chain is missing, the machine shows zero parts cost and GM% is overstated.
Consumables only cover Toner Delivery cases. Drums, waste toner, fusers and other consumables are excluded. Machines that consume high-cost consumables will appear more profitable than they are.
Labour rate is flat across all territories. NZD 49/hr is a proxy. Samoa technicians may cost more or less than Vanuatu technicians. Territory-specific rates should be applied in a later release.
VUV revenue scale. BMS stores VUV as full integer Vatu (not sub-units). However, the service rate tables (billing_contract_rates) use a "cents" convention even for VUV. The scale interpretation for per-click rates in VUV has not been fully validated against a known invoice. Revenue figures for Vanuatu machines should be cross-checked against a recent BMS invoice for one machine.
Data is static at build time. With the exception of the Actions tracker and QBR section, all dashboard data reflects the state when the files were last regenerated. The as-of date is shown in the header. Rebuild the files monthly (or when a significant contract change occurs) to keep data current.
NZD/RSP territory has only 3 machines. NZD-currency BMS contracts are few; most NZ-based MPS may be managed outside BMS or under different contract structures. Confirm with Vijendra whether NZ MPS machines are fully captured.
